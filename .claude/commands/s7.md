Génère le rapport S7 pour l'actif : $ARGUMENTS

1. **Si aucune adresse n'est fournie, demande-la avant tout** (AskUserQuestion).
2. Dépendances (n'installe que si l'import échoue) :
   `python3 -c "import reportlab, PIL, staticmap, playwright" 2>/dev/null || pip install reportlab pillow staticmap playwright`
3. Lance :
   `python3 skills/s6-biodiversite/scripts/s7_auto.py "<adresse>" --out S7_<actif>.pdf [--locataire "Nom"]`
4. Barème : au moins 2 modes d'acheminement différents (2 modes, ou desserte
   bus multi-lignes) → 3/3, sinon 0/3. Si la sortie parle de serveurs
   saturés, relance après ~1 min — ne conclus jamais 0 là-dessus.
5. Envoie le PDF tel quel (pas de re-vérification) en relayant le verdict,
   les modes/lignes et les contrôles.
6. **Contrôle IA (si présent dans la sortie)** : le script imprime une ligne
   « Contrôle IA : … » seulement si une clé OpenAI (`OPENAI_API_KEY`) est
   configurée. **Le script ne modifie jamais le score ; toute modification
   passe par l'accord explicite de l'utilisateur.**
   - « confirmé » → le mentionner brièvement.
   - « DOUTE — <raison> » :
     - Si un « Remplacement RÉEL proposé : X → Y » est affiché → proposer de
       remplacer l'arrêt X par Y (le 3/3 est conservé). **Seulement si
       l'utilisateur accepte**, relancer avec `--force-stops "A|Y"` (les arrêts
       gardés + le remplaçant Y).
     - Sinon (aucun remplaçant valable) → demander s'il faut baisser à 0/3 ;
       si oui, relancer avec `--force-score 0 --override-note "<raison>"`.
     - Ne jamais proposer un arrêt hors des candidats réels du script :
       aucune invention.
   - « indisponible » → le dire en une ligne, sans bloquer.
   - Aucune ligne « Contrôle IA » → pas de clé : comportement normal.
